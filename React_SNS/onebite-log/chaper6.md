# 6.1 인증 & 인가

사용자의 권한을 체크하는 행위

### 인증 Authentication

: 로그인 등의 수단으로 사용자의 신원을 확인하는 과정

### 인가 Authorization

: 인증된 사용자의 특정 데이터에 접근할 권한을 확인받는 과정

## 웹 서비스에서의 인증

### 1. 세션 방식 (사용자의 실시간성)

세션 객체를 통해 인증을 처리하는 방식

> 장점 : 세션 객체를 이용해, 사용자들의 상태를 관리할 수 있음
> 단점 : 메모리를 과하게 사용, 데이터베이스를 자주 조회해야 함

### 2. 토큰 방식

토큰을 이용해 인증을 처리하는 방식

> 장점 : 데이터베이스 조회 필요 x
> 단점 : 사용자들의 상태를 관리할 수 없음, 토큰 탈취 시 대처가 불가능 함

- JWT (Json Web Token)
  HEADER, PAYLOAD, SIGNATURE로 구성

- Access Token
  - 유효기간 설정: 30분 ~ 1시간
- Refresh Token
  - 유효기간 설정: 2주 ~ 1달
